# Lab 01: พื้นฐาน Git

ในแลปนี้ คุณจะได้ฝึกปฏิบัติการทำงานพื้นฐานของ Git ได้แก่ การสร้าง repository, การจัดเตรียมไฟล์ (staging), การตรวจสอบสถานะ, การตรวจสอบความแตกต่าง และการบันทึกการเปลี่ยนแปลง นอกจากนี้เราจะเรียนรู้วิธีการละเว้นไฟล์บางไฟล์โดยใช้ `.gitignore`

**ข้อกำหนดเบื้องต้น**: ต้องติดตั้ง Git บนเครื่องของคุณแล้ว

---

## คำแนะนำการทำแลป

### ขั้นตอนที่ 1: การตั้งค่าเริ่มต้น

1.  **สร้างไดเรกทอรีสำหรับทำงาน**:
    เปิด terminal และรันคำสั่งต่อไปนี้เพื่อสร้างไดเรกทอรีสำหรับแลปนี้:
    ```bash
    mkdir git-lab
    cd git-lab
    ```

2.  **เริ่มต้น Git**:
    สร้าง Git repository ใหม่ในไดเรกทอรีนี้:
    ```bash
    git init
    ```
    *คำสั่งนี้จะสร้างโฟลเดอร์ซ่อน `.git` ที่ใช้ติดตามการเปลี่ยนแปลงของคุณ*

### ขั้นตอนที่ 2: ขั้นตอนการทำงานพื้นฐาน (แก้ไข, เพิ่ม, บันทึก)

1.  **สร้างไฟล์**:
    สร้างไฟล์ข้อความสามไฟล์พร้อมเนื้อหา:
    ```bash
    echo "Content for file 1" > file1.txt
    echo "Content for file 2" > file2.txt
    echo "Content for file 3" > file3.txt
    ```

2.  **ตรวจสอบสถานะ**:
    ดูว่า Git มองไฟล์ใหม่เหล่านี้อย่างไร:
    ```bash
    git status
    ```
    *คุณจะเห็นไฟล์เหล่านี้แสดงอยู่ใน "Untracked files" (ไฟล์ที่ยังไม่ถูกติดตาม)*

3.  **เพิ่มไฟล์เข้า Staging Area**:
    เพิ่มไฟล์เข้าไปใน "Staging Area" ซึ่งเป็นการบอก Git ว่าคุณต้องการรวมการเปลี่ยนแปลงเหล่านี้ในการบันทึกครั้งถัดไป
    ```bash
    git add file1.txt file2.txt file3.txt
    ```

4.  **บันทึก (Commit)**:
    บันทึก snapshot พร้อมข้อความอธิบาย:
    ```bash
    git commit -m "Initial commit with three files"
    ```

### ขั้นตอนที่ 3: การแก้ไขไฟล์และการใช้ `git diff`

1.  **แก้ไขไฟล์**:
    เพิ่มเนื้อหาใหม่เข้าไปในสองไฟล์:
    ```bash
    echo "Additional content for file 1" >> file1.txt
    echo "Additional content for file 2" >> file2.txt
    ```

2.  **ตรวจสอบความแตกต่าง**:
    ก่อนที่จะเพิ่มไฟล์ เป็นการดีที่จะตรวจสอบว่ามีอะไรเปลี่ยนแปลงไปบ้าง:
    ```bash
    git diff
    ```
    *คำสั่งนี้จะแสดงความแตกต่างแบบทีละบรรทัดระหว่าง working directory ของคุณกับ commit ล่าสุด*

3.  **บันทึกทีละขั้นตอน**:
    มาแก้ไข staging area และบันทึกกันเลย
    ```bash
    git add file1.txt file2.txt
    git commit -m "Updated file1 and file2"
    ```

### ขั้นตอนที่ 4: การละเว้นไฟล์ด้วย `.gitignore`

บางครั้งคุณมีไฟล์ที่ไม่ต้องการติดตาม (ไฟล์ชั่วคราว, ไฟล์ build, ข้อมูลลับ)

1.  **สร้างไฟล์ "ลับ"**:
    ไฟล์เหล่านี้จำลองไฟล์ที่ไม่ควรอยู่ใน Git
    ```bash
    echo "Secret API Key" > .env
    echo "Debug logs..." > debug.log
    ```

2.  **สร้าง `.gitignore`**:
    สร้างไฟล์พิเศษชื่อ `.gitignore` และระบุ pattern ของไฟล์ที่ต้องการละเว้น:
    ```bash
    echo ".env" > .gitignore
    echo "*.log" >> .gitignore
    ```

3.  **ตรวจสอบสถานะ**:
    ตรวจสอบสถานะอีกครั้ง คุณจะเห็น `.gitignore` เป็นไฟล์ใหม่ที่ยังไม่ถูกติดตาม แต่จะ**ไม่**เห็น `.env` หรือ `debug.log`
    ```bash
    git status
    ```

4.  **บันทึกครั้งสุดท้าย**:
    เพิ่มไฟล์ ignore และการเปลี่ยนแปลงอื่นๆ ที่ค้างอยู่
    ```bash
    echo "More content" >> file3.txt
    git add .
    git commit -m "Final update with gitignore"
    ```

### ขั้นตอนที่ 5: ทบทวนประวัติ

ดูประวัติของโปรเจกต์ของคุณ:

```bash
git log --oneline --graph --all
```

## สรุปแนวคิดสำคัญ

### คำสั่ง Git ที่ใช้ในแลป

| คำสั่ง | คำอธิบาย |
|--------|----------|
| `git init` | เริ่มต้น repository ใหม่ |
| `git status` | ตรวจสอบสถานะของไฟล์ |
| `git add <file>` | เพิ่มไฟล์เข้า staging area |
| `git commit -m "message"` | บันทึก snapshot พร้อมข้อความ |
| `git diff` | แสดงความแตกต่างที่ยังไม่ได้ stage |
| `git log` | แสดงประวัติ commit |

### สถานะของไฟล์ใน Git

1. **Untracked** (ยังไม่ถูกติดตาม): ไฟล์ใหม่ที่ Git ยังไม่รู้จัก
2. **Modified** (ถูกแก้ไข): ไฟล์ที่เปลี่ยนแปลงแต่ยังไม่ได้ stage
3. **Staged** (พร้อมบันทึก): ไฟล์ที่อยู่ใน staging area รอการ commit
4. **Committed** (บันทึกแล้ว): ไฟล์ที่ถูกบันทึกใน repository แล้ว

### ทำไมต้องใช้ `.gitignore`?

- ป้องกันไม่ให้ข้อมูลลับ (API keys, passwords) ถูกอัปโหลด
- ไม่ติดตามไฟล์ที่สร้างโดยระบบ (logs, cache)
- ทำให้ repository สะอาดและมีเฉพาะ source code ที่จำเป็น
